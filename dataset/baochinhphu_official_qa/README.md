# BaoChinhPhu Official Q&A Dataset

This dataset contains official citizen and business Q&A articles from
`https://baochinhphu.vn/tra-loi-cong-dan.htm`.

The dataset is used to evaluate Vietnamese legal RAG and recommendation behavior. It is not a legal-document corpus by itself. Each record is an official answer article that can be used as ground truth for whether the model retrieves relevant law, cites the right provisions, and avoids unsupported legal advice.

Only articles with direct links to legal documents are included. A direct legal-document link currently means a `vanban.chinhphu.vn` URL with a `docid` query parameter inside the article body. At least one linked document must match the local Law Assistant legal-document database. Unmatched linked documents are not emitted in the usable record, so they cannot become ground truth for later evaluation or fine-tuning.

## Folder Layout

```text
baochinhphu_official_qa/
├── data_raw/      # Unchanged source captures, when retained for audit
├── data_usable/   # Clean JSON/JSONL files used by tests and evaluators
└── creation/      # Dataset builders, creation docs, and validation notes
```

Generated evaluation files should go in `data_usable/`. Raw HTML, listing responses, or source manifests should go in `data_raw/` if they are retained.

## Usable Record Contract

The main usable output is expected at:

```text
data_usable/rag_test_set.json
```

Each record should include these core fields:

- `question`: factual situation plus the citizen/business question.
- `expected_answer`: official answer text extracted from the article.
- `expected_legal_citations`: structured legal citations found in the official answer.
- `expected_citation_text`: newline-joined citation text for deterministic checks.
- `source_url`, `source_name`, `source_dataset`, and `article_id`: source traceability.
- `published_date`, `question_date`, and `retrieval_cutoff_date`: dates used for temporal evaluation.
- `tags` and `category`: source metadata useful for filtering and analysis.
- `direct_legal_document_links`: direct legal-document links that matched the local database.
- `matched_direct_legal_documents`: linked documents that matched the local database.

`expected_legal_citations` should contain only citations that correspond to matched local documents. Provision and `citation_text` come from the official answer text; document metadata comes from the local database:

- `provision`, for example `Khoản 1 Điều 72`.
- `citation_text`, for example `Khoản 1 Điều 72 Quyết định số 505/QĐ-BHXH ngày 27/3/2020`.
- `document_id`, `document_name`, `document_type`, `document_number`, `document_date`, and `document_status` from the matched local legal-document metadata.

These citation fields are extracted from the official answer. They are not generated by the model.

## Evaluation Use

Use this dataset for:

- Retrieval checks: did the service retrieve legal chunks that cover the official cited documents and provisions?
- Citation checks: did the answer cite legal document names and clauses, not only generic references?
- Groundedness checks: did the answer stay within retrieved/legal source support?
- Recommendation checks: did similar Q&A articles surface for related legal situations?

For legal safety, a model answer should be penalized when it gives a legal conclusion without source-backed citations or when retrieved context does not support the cited provision.

Related article links from BaoChinhPhu are used only to discover more candidate pages during crawling. They are not saved as dataset ground truth.
